# On My Way Homepage Plan

## 1. Current Phase

현재 단계는 홈페이지 제작 전 설계 단계다.

이번 단계의 산출물:

- `docs/spec.md`: 서비스와 홈페이지의 제품 명세
- `docs/plan.md`: 설계, 제작, 검증, 출시 계획

## 1.1 Confirmed Decisions

- 서비스명은 `On My Way`로 확정한다.
- 첫 타깃은 일정 기간 안에 목표를 설정하고 달성하고 싶어 하는 사람들이다.
- 메인 CTA는 유저의 성향을 파악한 뒤, 그에 맞는 목표 설정 방법을 제안하는 진단 시작이다.
- 성향 파악 입력값은 사주/만세력 정보, MBTI, 목표 유형, 목표 기간, 현재 상태를 우선 검토한다.
- 컬러는 `#1F41B0`, `#F4F4F4`, `#E34054` 조합을 사용한다.
- 폰트는 `S-Core Dream`을 사용한다.
- 성장 테마는 식물, 강아지, 고양이, 집짓기로 확정한다.
- Pricing은 무료 진단, 하루 체험, 월 구독, 기간형 목표 패스 구조로 설계한다.

## 2. Working Assumptions

현재 확정값과 아직 남은 가정은 다음과 같다.

- 초기 시장은 한국어 사용자다.
- 첫 화면은 제품 소개보다 "성향 기반 목표 설계" 경험을 우선한다.
- 초기 수익화는 무료 결과 미리보기 후 하루 체험, Premium Monthly, Goal Pass 전환으로 검증한다.
- 실제 코칭 기능은 후속 확장으로 두고, MVP는 자동 계획과 체크인에 집중한다.
- 홈페이지는 신뢰감 있는 생산성 앱과 자기 이해 기반 코칭 서비스의 중간 톤으로 만든다.
- 사주/만세력과 MBTI는 목표 달성을 보장하는 판단 기준이 아니라 목표 설정 스타일을 찾는 참고 데이터로 다룬다.

## 4. Homepage Build Plan

### Section 1. Header

작업:

- 로고 자리 확보
- 핵심 메뉴 구성
- 로그인 버튼
- primary CTA 버튼
- 모바일 메뉴 구현

완료 기준:

- 데스크톱과 모바일에서 주요 이동이 명확하다.
- CTA가 항상 쉽게 발견된다.

### Section 2. Hero

작업:

- H1, supporting copy 작성
- 목표 입력 필드 구현
- MBTI/만세력 선택 입력 진입점 구현
- primary CTA 구현
- product preview 배치
- 모바일 첫 화면 최적화

완료 기준:

- 방문자가 5초 안에 서비스의 역할을 이해한다.
- 사용자가 자신의 목표와 성향 정보를 입력해볼 수 있다.

### Section 3. Problem

작업:

- "남들이 좋다는 방식이 내 성향에는 맞지 않았다"를 핵심 문제로 정리
- 보조 실패 패턴 3개를 함께 구성
- 짧고 구체적인 카피 작성
- 시각적으로 빠르게 스캔 가능한 레이아웃 구현

완료 기준:

- 사용자 문제를 비난하지 않고 정확히 짚는다.
- 문제 섹션이 성향 기반 해결 방식으로 자연스럽게 이어진다.

### Section 4. Personality-Based Solution

작업:

- 성향 파악, 목표 설정 방식 추천, 계획 생성, 체크인/조정 4단계 설명
- 각 단계에 맞는 UI 또는 아이콘 구성
- 다음 섹션의 제품 미리보기로 자연스럽게 연결

완료 기준:

- On My Way가 성향 정보를 어떻게 목표 설정 방식으로 바꾸는지 추상적이지 않다.

### Section 5. Product Preview

작업:

- 목표 대시보드 mock
- 성향 분석 요약 mock
- 추천 목표 설정 방식 mock
- 오늘 할 일 mock
- 주간 진행률 mock
- 목표 달성 게이지 mock
- 하루 스케줄 체크 mock
- 전체 스케줄 PDF 다운로드 mock
- 계획 조정 알림 mock
- 성장 보상 mock

완료 기준:

- 실제 제품을 써보고 싶다는 느낌을 준다.

### Section 6. Growth Companion

작업:

- 하루 목표 완료 시 성장 보상 흐름 설계
- 성장 테마 구성: 식물, 강아지, 고양이, 집짓기
- 하루 체험에서 성장 대상 설정, 이름 붙이기, 첫 유대감 형성 플로우 설계
- 결제 후 목표 기간 기반 일일 성장 단계 설계
- 최종 목표 달성 시 성장 대상의 완성 모습 설계
- 성장 대상 응원 메시지 톤 설계
- 완료/일부 완료/미완료 상태별 보상 문구 설계
- 모바일에서 귀엽지만 과하지 않은 성장 미리보기 UI 설계
- 프리미엄 전환 가치로 일일 성장 여정, 성장 기록 저장, 장기 성장 컬렉션을 연결

완료 기준:

- 하루의 실행이 눈에 보이는 성장으로 이어진다는 느낌이 든다.
- 하루 체험만으로도 성장 대상과의 유대감이 생긴다.
- 성장 보상이 목표 관리의 본질을 가리지 않고 체크인 동기를 강화한다.

### Section 7. Use Cases

작업:

- 목표 유형별 예시 작성
- 각 유형의 계획 예시 작성
- 6개 카테고리 확정: 시험/자격증, 운동/다이어트, 취업/이직, 부업/프로젝트, 습관 형성, 저축/돈 관리
- 모바일에서 가로 스크롤 또는 탭 UI 검토

완료 기준:

- 사용자가 자신의 목표 유형을 찾을 수 있다.

### Section 8. Pricing

작업:

- Free/Premium/Coaching 구조 구성
- Free Diagnosis, One-Day Trial, Premium Monthly, Goal Pass, Coaching 구조 구성
- 무료 미리보기와 유료 앱 진행 범위 구분
- 초기에는 정확한 가격 대신 하루 체험과 프리미엄 전환 가설 중심으로 표현
- 결제 후 하루 스케줄 체크, 목표 달성 게이지, PDF 다운로드 기능 강조
- 결제 CTA 배치

완료 기준:

- 무료 시작의 부담이 낮고, 유료 전환의 이유가 보인다.
- 유료 전환이 강압적인 잠금이 아니라 목표 달성을 계속 진행하는 흐름으로 느껴진다.
- 월 구독과 기간형 목표 패스 중 유저에게 맞는 결제 선택지가 보인다.

추천 가격안:

- Free Diagnosis: 무료
- One-Day Trial: 초기에는 무료, 이후 1,000원 이하 유료 체험 A/B 테스트
- Premium Monthly: 월 9,900원
- Goal Pass: 30일 14,900원 / 90일 39,000원 / 100일 44,000원
- Coaching: 추후 별도 책정

### Section 9. Final CTA

작업:

- 목표 입력 재노출
- 짧은 마무리 카피 작성
- CTA 연결

완료 기준:

- 페이지를 다 읽은 사용자가 다시 행동할 수 있다.

### Admin Section. Operations Dashboard

작업:

- 관리자 Overview dashboard 설계
- 유저 통계 카드 구성: 가입자, 활성 유저, 유료 유저, 해지율
- 목표 통계 구성: 목표 생성 수, 목표 유형별 분포, 목표 달성률, 중도 포기율
- 퍼널 통계 구성: 목표 입력, 성향 진단, 결과 미리보기, 하루 체험, 결제 전환
- 사용 통계 구성: 체크인 완료율, 성장 보상 확인률, PDF 다운로드율, 목표 달성 게이지 확인률
- 리텐션 통계 구성: D1, D7, D30 리텐션과 이탈 위험 구간
- 필터 구성: 날짜, 목표 유형, 성향 추천 방식, 결제 상태, 체험 상태
- 운영자가 유지보수할 수 있도록 목표 템플릿, 성장 테마, 추천 문구 관리 영역 설계

완료 기준:

- 운영자가 유저 상황과 목표 달성률을 한눈에 파악할 수 있다.
- 목표 유형별 문제 구간과 이탈 구간을 찾을 수 있다.
- 기능 사용률과 유지율의 관계를 확인할 수 있다.
- 민감 정보는 원본 열람보다 집계 통계 중심으로 다룬다.

- 목표 기간 기반 일일 성장 단계 생성
- 성장 대상 응원 메시지 생성

## 7. Content Plan

초기 제작에 필요한 콘텐츠:

- 브랜드명
- 로고 또는 임시 워드마크
- hero headline
- 목표 입력 placeholder
- 성향 입력 안내 문구
- MBTI 선택 문구
- 만세력 입력 안내 및 개인정보 안내 문구
- 문제 섹션 카피
- 4단계 솔루션 카피
- 기능 설명
- 가격 플랜 설명
- 결제 유도 문구
- 하루 체험 안내 문구
- 결제 후 스케줄 체크 안내 문구
- 목표 달성 게이지 안내 문구
- 전체 스케줄 PDF 안내 문구
- 성장 보상 안내 문구
- 개인정보/신뢰 문구
- CTA 문구

콘텐츠 작성 원칙:

- 사용자의 의지 부족을 탓하지 않는다.
- "성공 보장"처럼 검증되지 않은 약속은 피한다.
- 사주/만세력, MBTI를 결정론처럼 표현하지 않는다.
- 성향 분석은 자기 이해와 계획 설계를 돕는 참고 도구로 표현한다.
- 목표 달성을 작은 실행과 조정의 반복으로 설명한다.
- 유료 전환은 압박이 아니라 더 강한 지원으로 설명한다.
- 무료 결과 미리보기 이후 결제 유도는 "목표 달성을 앱에서 계속 진행"하는 표현을 사용한다.

## 8. Validation Checklist

디자인 검수:

- 첫 화면에서 서비스 목적이 보이는가?
- 실제 제품 화면 또는 그에 준하는 preview가 있는가?
- 모바일에서 목표 입력이 편한가?
- 모바일에서 MBTI/만세력 입력이 부담스럽지 않은가?
- CTA가 충분히 명확한가?
- 색상과 타이포그래피가 한쪽 톤으로 과하게 치우치지 않았는가?

제품 검수:

- 목표 입력 후 다음 행동이 분명한가?
- 성향 입력 후 어떤 결과를 받는지 분명한가?
- 사주/만세력과 MBTI가 목표 설계에 어떻게 쓰이는지 설명되는가?
- 무료 사용자와 유료 사용자의 차이가 이해되는가?
- 결과 미리보기 이후 결제 유도 이유가 납득되는가?
- 결제 전환 CTA가 너무 이르거나 강압적으로 느껴지지 않는가?
- 결제 후 앱에서 무엇을 하게 되는지 하루 스케줄 체크 화면으로 명확히 보이는가?
- 목표 달성 게이지가 유저에게 진행감을 주는가?
- 전체 스케줄 PDF가 결제 후 산출물로 충분히 가치 있게 느껴지는가?
- 하루 목표 완료 후 성장 보상이 목표 실행 동기와 자연스럽게 연결되는가?
- 최종 성장 모습이 목표 기간에 맞춰 자연스럽게 분할되는가?
- 하루 체험에서 성장 대상과의 유대감이 충분히 형성되는가?
- 성장 보상이 너무 유치하거나 목표 관리 본질을 가리지 않는가?
- 사용자가 기대할 기능과 홈페이지 설명이 어긋나지 않는가?

사업 검수:

- 유료 전환 이유가 보이는가?
- 하루 체험 또는 프리미엄 결제 전환을 측정할 수 있는가?
- PDF 다운로드율과 목표 달성 게이지 확인률을 측정할 수 있는가?
- 성장 보상이 체크인 유지율에 영향을 주는지 측정할 수 있는가?
- 성장 대상 설정 완료율과 일일 성장 확인률을 측정할 수 있는가?
- 관리자 화면에서 유저 상황, 목표 달성률, 이탈 구간을 확인할 수 있는가?
- 운영자가 목표 유형별 문제와 기능별 유지율 영향을 파악할 수 있는가?
- 초기 실험 지표를 측정할 수 있는가?
- 베타 사용자 피드백을 받을 수 있는가?
