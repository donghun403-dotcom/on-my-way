# On My Way Homepage Spec

> **과거 설계 문서:** 이 문서의 가격·체험·상품 아이디어는 현재 운영 정책이 아닙니다. 현재 확정 정책은 [`pricing-and-credits.md`](pricing-and-credits.md)만 기준으로 사용합니다.

## 1. Project Summary

On My Way는 사용자가 일정 기간 안에 이루고 싶은 목표를 구체화하고, 본인의 성향에 맞는 방식으로 실행 계획을 세우고, 진행 상황을 함께 체크하며 목표 달성 확률을 높이도록 돕는 서비스다.

홈페이지의 목적은 단순 소개가 아니라, 사용자가 "내 성향에 맞는 목표 설정 방법을 여기서 찾을 수 있겠다"는 확신을 갖고 성향 진단, 목표 입력, 회원가입, 무료 체험, 유료 전환 중 하나의 행동으로 이동하게 만드는 것이다.

## 1.1 Confirmed Direction

- Service name: On My Way
- Primary target: 일정 기간 안에 목표를 설정하고 달성하고 싶어 하는 사람들
- Main CTA: 유저의 성향을 파악한 뒤 그에 맞는 목표 설정 방법을 제안하는 진단 시작
- Personality inputs: 사주/만세력 정보, MBTI, 목표 유형, 기간, 현재 상태
- Positioning: 성향 기반 목표 설정과 실행 체크를 결합한 목표 달성 파트너
- Brand colors: #1F41B0, #F4F4F4, #E34054
- Brand font: S-Core Dream
- Growth themes: 식물, 강아지, 고양이, 집짓기

## 2. Business Goal

서비스의 최종 목적은 사용자의 목표 달성을 돕는 과정에서 지속 가능한 이윤을 창출하는 것이다.

핵심 수익 모델은 다음 순서로 검증한다.

1. 성향 기반 무료 목표 진단과 결과 미리보기 후 프리미엄 플랜 전환
2. 월 구독 기반의 목표 관리 코칭
3. 특정 기간형 목표 프로그램 판매
4. 고관여 사용자를 위한 1:1 또는 그룹 코칭 연계
5. 기업/팀용 목표 관리 상품 확장

## 3. Target Users

### Primary User

일정 기간 안에 달성하고 싶은 목표가 있고, 본인에게 맞는 목표 설정 방식과 실행 루틴을 찾고 싶은 개인 사용자.

예시:

- 시험, 자격증, 취업, 이직을 준비하는 사람
- 운동, 다이어트, 건강 루틴을 만들고 싶은 사람
- 부업, 창업, 콘텐츠 제작처럼 긴 호흡의 프로젝트를 시작하려는 사람
- 독서, 영어, 포트폴리오, 저축 등 자기계발 목표를 지속하고 싶은 사람
- MBTI, 사주/만세력, 성향 진단처럼 자기 이해 도구에 관심이 있는 사람

### Secondary User

목표 관리가 필요한 소규모 팀, 스터디, 코칭 그룹.

초기 홈페이지는 개인 사용자에게 집중하고, 팀/그룹 기능은 후속 확장 영역으로만 노출한다.

## 4. Core User Problem

사용자는 목표 자체보다 목표를 지속 가능한 계획으로 바꾸는 과정에서 자주 실패한다.

주요 문제:

- 목표가 추상적이다.
- 기간과 실행량이 현실적이지 않다.
- 오늘 무엇을 해야 하는지 명확하지 않다.
- 본인의 성향과 맞지 않는 계획 방식을 반복한다.
- 계획을 세우는 방식이 너무 일반적이라 개인에게 맞지 않는다.
- 계획이 틀어졌을 때 다시 조정하지 못한다.
- 혼자 하다 보니 체크와 피드백이 끊긴다.
- 성취감이 늦게 오기 때문에 중간에 포기한다.

## 5. Product Promise

서비스가 전달해야 할 핵심 약속:

"당신의 성향을 이해한 뒤, 목표를 세우는 방식부터 실행하고 점검하는 방식까지 맞춰서 함께 간다."

짧은 대체 문구:

- 내 성향에 맞는 목표 설정, 끝까지 가는 실행 관리
- 목표를 계획으로, 계획을 나에게 맞는 실행으로
- 혼자 흐려지는 목표를 On My Way가 끝까지 선명하게
- 오늘 할 일을 놓치지 않게 도와주는 성향 기반 목표 달성 파트너

## 6. Homepage Objective

홈페이지는 다음 역할을 수행해야 한다.

1. 사용자의 문제를 빠르게 짚는다.
2. On My Way가 성향 분석을 통해 목표 설정 방식을 어떻게 다르게 제안하는지 보여준다.
3. 성향 진단과 목표 입력으로 즉시 시작하게 만든다.
4. 무료 사용과 유료 전환의 이유를 설득한다.
5. 신뢰를 만든다.

## 7. Conversion Funnel

### Main Funnel

1. 방문
2. 성향 기반 목표 설계 CTA 클릭
3. 목표 유형, 목표 기간, 현재 상태 입력
4. MBTI, 생년월일/출생시간/출생지 등 만세력 분석에 필요한 정보 선택 입력
5. 성향에 맞는 목표 설정 방식과 계획 미리보기 제공
6. 앱에서 전체 목표 설계와 체크인을 계속하려면 결제 또는 하루 체험 시작 유도
7. 이메일/소셜 회원가입
8. 프리미엄 플랜 또는 체험 플랜 활성화
9. 전체 기간 목표 계획 생성
10. 체크인 반복
11. 유료 유지 또는 코칭 플랜 업셀

### Secondary Funnel

1. 방문
2. 기능/가격 확인
3. 사례 또는 후기 확인
4. 무료 체험 시작

## 8. Homepage Information Architecture

### 8.1 Header

목적:

- 브랜드 인지
- 주요 페이지 이동
- 즉시 시작 CTA 제공

구성:

- Logo
- Product
- How it works
- Personality Plan
- Pricing
- Stories
- Login
- Primary CTA: 내 성향에 맞는 목표 설계하기

### 8.2 Hero

목적:

- 첫 화면에서 서비스 정체성과 사용자의 다음 행동을 명확히 한다.

권장 구성:

- H1: On My Way, 내 성향에 맞게 목표를 세우고 끝까지 가는 길
- Supporting copy: MBTI와 만세력 기반 성향 입력, 목표 기간, 현재 상황을 바탕으로 나에게 맞는 목표 설정 방식과 실행 계획을 제안합니다.
- Goal input field: 예) "3개월 안에 토익 900점 달성"
- Personality input entry: MBTI 선택, 생년월일/출생시간/출생지 입력 선택
- CTA: 내 성향에 맞는 목표 설계하기
- Secondary CTA: 작동 방식 보기
- Product preview: 성향 요약, 목표 설정 방식, 주간 계획, 오늘 체크인, 진행률이 보이는 실제 UI 이미지 또는 인터랙티브 미리보기

주의:

- 추상적인 일러스트보다 실제 제품 화면을 첫 화면에 보여준다.
- 방문자가 첫 화면에서 바로 본인의 목표를 입력해볼 수 있어야 한다.
- 성향 정보는 목표 입력 후 다음 단계에서 선택 입력으로 받는다.
- 사주/만세력과 MBTI는 운명 결정이나 성공 보장처럼 표현하지 않고, 목표 설정 스타일을 찾기 위한 참고 데이터로 설명한다.

### 8.3 Problem Section

목적:

- 사용자가 "내 얘기다"라고 느끼게 한다.

내용 방향:

- 목표를 세운 날은 분명했지만, 일주일 뒤에는 흐려짐
- 계획이 너무 크거나 막연해서 오늘 할 일이 보이지 않음
- 실패한 날 이후 계획 전체가 무너짐
- 중간 점검이 없어서 늦게 알아차림
- 핵심 메시지: 남들이 좋다는 방식으로 해봤지만, 내 성향에는 잘 맞지 않았다.

### 8.4 Solution Section

목적:

- 서비스의 도움 방식을 4단계로 설명한다.

구성:

1. 성향 파악
   - MBTI, 만세력 기반 정보, 목표 유형, 현재 상태를 바탕으로 사용자의 계획 성향을 정리한다.
2. 목표 설정 방식 제안
   - 몰입형, 루틴형, 보상형, 점검형 등 사용자에게 맞는 목표 설정 방식을 제안한다.
3. 실행 계획 생성
   - 월간, 주간, 일간 단위로 나누고 오늘 해야 할 일을 제시한다.
4. 체크인과 조정
   - 완료 여부, 어려움, 누락된 일정을 바탕으로 계획을 다시 조정한다.

### 8.5 Product Preview Section

목적:

- 사용자가 실제로 어떤 경험을 하게 되는지 보여준다.

노출할 화면:

- 목표 대시보드
- 성향 분석 요약
- 추천 목표 설정 방식
- 오늘의 체크리스트
- 주간 진행률
- 계획 조정 알림
- 목표 회고
- 결제 후 앱에서 이어지는 전체 기간 계획과 체크인 안내
- 하루 목표 완료 시 성장 보상 미리보기

무료 미리보기 범위:

- 성향 요약
- 추천 목표 설정 방식
- 이번 주 실행 계획 일부
- 오늘의 첫 행동
- 체크/조정 방식 맛보기

결제 후 제공 범위:

- 전체 기간 목표 계획
- 앱 내 목표 저장
- 데일리 체크인
- 오늘의 스케줄 체크 화면
- 목표 달성 게이지
- 목표 기간에 맞춘 일일 성장 단계
- 하루 목표 완료 시 성장 대상의 변화
- 성장 대상과 주고받는 응원 메시지
- 전체 스케줄 PDF 다운로드
- 주간 회고 리포트
- 놓친 계획 재조정
- 진행률 관리

결제 후 앱 경험:

- 유저는 매일 생성된 하루 스케줄을 체크하면서 목표를 이행한다.
- 하루 스케줄은 시간대, 할 일, 예상 소요 시간, 완료 체크를 포함한다.
- 목표 달성 게이지는 전체 목표 기간 대비 현재 완료율을 보여준다.
- 게이지는 퍼센트, 남은 기간, 현재 페이스를 함께 보여줄 수 있다.
- 하루 계획을 모두 이행하면 성장 대상의 해당 날짜 성장 모습이 열린다.
- 전체 스케줄 PDF는 결제 후 다운로드할 수 있는 산출물로 제공한다.
- PDF에는 목표 요약, 성향 기반 목표 설정 방식, 전체 기간 계획, 주간 계획, 일일 체크리스트가 포함된다.

### 8.6 Growth Companion Section

목적:

- 장기 목표의 느린 성취감을 매일 눈에 보이는 성장으로 보완한다.
- 유저가 하루 목표를 완수했을 때 감정적 보상을 제공한다.
- 체크인을 기록 행위가 아니라 귀여운 성장 경험으로 만든다.
- 성장 대상과 함께 목표를 향해 가는 유대감을 형성한다.

성장 테마 예시:

- 식물 키우기
- 강아지 키우기
- 고양이 키우기
- 집짓기

작동 방식:

- 유저는 목표 시작 시 성장 테마를 선택한다.
- 유저는 성장 대상의 이름과 최종 목표 달성 시 완성 모습을 설정한다.
- 시스템은 목표 기간을 기준으로 최종 성장 모습을 하루 단위 성장 단계로 나눈다.
- 하루 목표를 완벽히 이행하면 해당 날짜의 성장 단계가 열린다.
- 성장 대상은 완료 메시지와 다음 날 응원 메시지로 유저와의 유대감을 만든다.
- 일부 완료 시에는 작은 변화, 유지 보상, 응원 메시지를 제공한다.
- 실패한 날에도 성장을 망가뜨리지 않고 다음 날 다시 이어갈 수 있게 한다.
- 장기 목표가 끝나면 완성된 성장 결과물을 저장하거나 공유할 수 있다.

성장 단계 설계:

- 최종 목표 달성일의 완성 모습을 먼저 정한다.
- 목표 기간이 30일이면 30개의 성장 단계를 만든다.
- 목표 기간이 90일 이상이면 큰 성장 단계와 미세 변화를 섞어 지루함을 줄인다.
- 성장은 목표 실행과 연결되어야 하며, 단순 출석 보상이 아니라 하루 계획 완료 보상이어야 한다.

수익화 연결:

- 무료 미리보기에서는 성장 테마 선택과 최종 성장 모습 예시를 보여준다.
- 하루 체험에서는 성장 대상 설정, 이름 붙이기, 첫 체크인, 첫 응원 메시지까지 경험하게 한다.
- 프리미엄에서는 전체 목표 기간에 맞춘 일일 성장 여정, 장기 성장 기록, 성장 컬렉션, 테마 추가, 목표별 성장 대상 저장을 제공한다.

### 8.8 Use Cases Section

초기 홈페이지에서 보여줄 목표 유형:

- 시험/자격증
- 운동/다이어트
- 취업/이직
- 부업/프로젝트
- 습관 형성

## 9. Admin Console Requirements

관리자 페이지는 유저들의 사용 상황을 통계적으로 확인하고, 서비스 유지보수와 개선 의사결정을 돕는 운영 대시보드다.

목적:

- 유저들이 어디에서 이탈하는지 확인한다.
- 어떤 목표 유형이 가장 많이 생성되고 잘 달성되는지 확인한다.
- 성향 진단, 하루 체험, 결제, 체크인, 성장 보상, PDF 다운로드가 실제 유지율에 어떤 영향을 주는지 확인한다.
- 목표 달성률과 이탈률을 기반으로 제품 개선 우선순위를 정한다.
- 고객 지원과 운영 대응에 필요한 유저 상태를 빠르게 파악한다.

- 만세력 정보 입력 비율
- 성향 진단 완료율

### 9.2 Admin Views

필수 화면:

1. Overview Dashboard
   - 가입, 활성, 결제, 리텐션, 목표 달성률을 요약한다.
2. User Analytics
   - 유저별 목표 수, 체크인 상태, 결제 상태, 최근 활동을 확인한다.
3. Goal Analytics
   - 목표 유형, 기간, 달성률, 중도 포기율, 평균 진행률을 확인한다.
4. Funnel Analytics
   - 목표 입력, 성향 진단, 결과 미리보기, 하루 체험, 결제 전환 흐름을 확인한다.
5. Engagement Analytics
   - 체크인, 성장 보상, 목표 달성 게이지, PDF 다운로드 사용률을 확인한다.
6. Retention and Churn
   - 이탈 위험 유저와 이탈 구간을 확인한다.
7. Content and Template Management
   - 목표 템플릿, 성장 테마, 성향별 추천 문구를 관리한다.

### 9.3 Maintenance Use Cases

관리자는 다음 질문에 답할 수 있어야 한다.

- 어떤 목표 유형에서 달성률이 가장 높은가?
- 어떤 목표 유형에서 중도 포기가 많은가?
- 하루 체험을 시작한 유저가 결제하지 않는 주요 구간은 어디인가?
- 성장 대상 설정을 완료한 유저의 리텐션이 더 높은가?
- PDF를 다운로드한 유저의 결제 유지율이 더 높은가?
- 특정 성향 추천 방식에서 실패율이 높지는 않은가?
- 어떤 성장 테마가 체크인을 가장 오래 유지시키는가?
- 어떤 목표 기간에서 계획 수정이 가장 많이 발생하는가?

### 9.4 Admin Design Direction

- 운영 도구는 귀여운 랜딩 톤보다 조용하고 실무적인 대시보드 톤으로 만든다.
- 큰 숫자, 추세 그래프, 목표 유형별 필터, 날짜 범위 필터를 우선한다.
- 목표 유형, 성향 유형, 결제 상태, 체험 상태, 기간별 필터를 제공한다.
- 유저 개인정보는 필요한 최소 범위만 보여주고, 사주/만세력 원본 정보는 운영자가 직접 열람하지 않는 방향을 우선한다.
- 민감 정보는 집계 통계 중심으로 사용한다.
